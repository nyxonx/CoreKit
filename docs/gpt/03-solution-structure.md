# Solution Structure

## Expected Layout

- `src/` sadrzi produkcione projekte
- `tests/` sadrzi test projekte
- `docs/` sadrzi tehnicku i produktnu dokumentaciju
- `.github/` sadrzi uputstva i automatizaciju za repozitorijum

## Naming

- Nazive projekata drzati kratkim i konzistentnim
- Test projekte imenovati prema projektu koji pokrivaju
- Dokumenta imenovati tako da se lako skeniraju i sortiraju

## Growth Rules

- Kada solution poraste, grupisati projekte po odgovornosti
- Izbegavati pretrpavanje root foldera
- Sve shared smernice drzati u `docs/` ili `.github/`
